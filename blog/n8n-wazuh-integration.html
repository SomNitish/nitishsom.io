Wait... Are We Actually Monitoring Everything?

Here's a question most SOC teams don't ask: Are you monitoring the system that responds to your security alerts?

We deployed multiple open-source security tools — Wazuh for SIEM, SentinelOne for EDR, Suricata for network monitoring, n8n for automation. Each tool had dashboards, alerts, and logging. Everything looked good on paper.

But here's what nobody talks about: We were monitoring our infrastructure, but not the automation layer itself.

Our n8n instance handled:

  • Phishing email triage
  • SentinelOne alert routing
  • Agent disconnection reporting
  • Cisco Meraki daily/weekly reports
  • JIRA ticketing automation
  • SOC hourly health reports

If a workflow silently started failing every day — nobody knew.
If an analyst manually triggered a sensitive workflow at 3 AM — no audit trail.
If n8n itself crashed and our entire automated response pipeline went dark — Wazuh had no idea.

We had a blind spot over the system that responds to our infrastructure alerts.

The Architecture: What We're Actually Monitoring

What is database.sqlite?

n8n stores ALL execution data in a single SQLite database file inside its Docker container:

/opt/n8n-server/YOUR_N8N_DATA_PATH/database.sqlite

This database contains:

  • execution_entity — Every workflow run (success/failure/running)
  • workflow_entity — Workflow definitions and metadata
  • user — User accounts and identity
  • project / project_relation / shared_workflow — Ownership chains

The entire execution history is right there. We just needed to read it and forward it to Wazuh.

Enabling SQLite in n8n (Docker)

By default, n8n uses SQLite when no database is configured. If you're running n8n in Docker, here's how to ensure SQLite persistence:

The Pipeline: Architecture Overview

The integration consists of 3 Python scripts that run continuously or on schedule, plus 1 cache builder:

  1. poller.py — Systemd daemon, polls SQLite every 15 seconds, enriches and forwards events
  2. heartbeat.py — Cron hourly, detects dead silence (no executions in 2hrs)
  3. summary.py — Cron 2x daily (10:00 & 22:00 IST), generates 12hr health reports
  4. workflow_map_builder.py — Cron every 48hrs, builds owner identity cache

Plus 3 state files:

  • last_id.txt — Cursor tracking (last processed execution ID)
  • seen_workflows.json — First-ever-seen detection for new workflows
  • workflow_map.json — Cached workflow → owner identity map (avoids expensive joins)
Why Multiple Scripts? poller.py handles real-time execution forwarding (every 15 seconds)
heartbeat.py solves the "absence detection" problem — event-driven systems can't detect silence
summary.py provides timed snapshots for night shift without requiring dashboard access
workflow_map_builder.py avoids expensive 5-table joins on every poll cycle

Script 1: poller.py — Core Collection Engine

Purpose: Runs continuously as a systemd service. Polls SQLite every 15 seconds, enriches execution records with IST timestamps, duration, owner identity, and new workflow detection. Writes JSON events to monitored log file.

Save as: /opt/YOUR_SCRIPTS_PATH/poller.py

#!/usr/bin/env python3
"""
n8n → Wazuh Integration: Poller Script
Continuously polls n8n's SQLite database and forwards enriched execution events to Wazuh.

BEFORE RUNNING: Update these path variables for your environment!
"""
import sqlite3, json, time, os
from datetime import datetime, timezone, timedelta

# ═══════════════════════════════════════════════════════════
# PATH CONFIGURATION — UPDATE THESE FOR YOUR ENVIRONMENT
# ═══════════════════════════════════════════════════════════

DB_PATH   = "/opt/n8n-server/YOUR_N8N_DATA_PATH/database.sqlite"  # ← UPDATE THIS
LOG_PATH  = "/var/log/n8n-executions.log"                          # ← UPDATE THIS
CURSOR_F  = "/opt/YOUR_SCRIPTS_PATH/last_id.txt"                   # ← UPDATE THIS
SEEN_WF_F = "/opt/YOUR_SCRIPTS_PATH/seen_workflows.json"           # ← UPDATE THIS
MAP_PATH  = "/opt/YOUR_SCRIPTS_PATH/workflow_map.json"             # ← UPDATE THIS

# ═══════════════════════════════════════════════════════════
# Script Configuration
# ═══════════════════════════════════════════════════════════

INTERVAL = 15  # Poll interval in seconds
IST      = timezone(timedelta(hours=5, minutes=30))

# ═══════════════════════════════════════════════════════════
# Helper Functions
# ═══════════════════════════════════════════════════════════

def load_cursor():
    try:
        return int(open(CURSOR_F).read().strip())
    except:
        return 0

def save_cursor(last_id):
    with open(CURSOR_F, "w") as f:
        f.write(str(last_id))

def load_seen_workflows():
    try:
        return json.load(open(SEEN_WF_F))
    except:
        return {}

def save_seen_workflows(seen):
    with open(SEEN_WF_F, "w") as f:
        json.dump(seen, f)

def load_workflow_map():
    try:
        return json.load(open(MAP_PATH))
    except:
        return {}

def parse_dt_utc(val):
    """Parse datetime string and attach UTC timezone"""
    if not val or val == "null":
        return None
    for fmt in ("%Y-%m-%d %H:%M:%S.%f", "%Y-%m-%d %H:%M:%S",
                "%Y-%m-%dT%H:%M:%S.%fZ", "%Y-%m-%dT%H:%M:%SZ"):
        try:
            return datetime.strptime(val, fmt).replace(tzinfo=timezone.utc)
        except:
            continue
    return None

def to_ist(dt_utc):
    """Convert UTC datetime to IST"""
    if dt_utc is None:
        return None
    return dt_utc.astimezone(IST)

def calc_duration(started, stopped):
    """Calculate execution duration in seconds"""
    s = parse_dt_utc(started)
    e = parse_dt_utc(stopped)
    if s and e:
        return round((e - s).total_seconds(), 2)
    return None

# ═══════════════════════════════════════════════════════════
# Main Polling Function
# ═══════════════════════════════════════════════════════════

def poll():
    """Poll SQLite for new executions and forward to Wazuh"""
    last_id        = load_cursor()
    seen_workflows = load_seen_workflows()
    workflow_map   = load_workflow_map()

    # Read-only SQLite connection
    con = sqlite3.connect("file:" + DB_PATH + "?mode=ro", uri=True)
    con.row_factory = sqlite3.Row
    rows = con.execute(
        "SELECT e.id, e.workflowId, w.name AS workflowName, "
        "e.status, e.mode, e.finished, e.startedAt, e.stoppedAt, e.retryOf "
        "FROM execution_entity e "
        "LEFT JOIN workflow_entity w ON w.id = e.workflowId "
        "WHERE e.id > ? ORDER BY e.id ASC",
        (last_id,)
    ).fetchall()
    con.close()

    if not rows:
        return

    with open(LOG_PATH, "a") as lf:
        for row in rows:
            # Calculate duration
            duration     = calc_duration(row["startedAt"], row["stoppedAt"])
            
            # Get timestamps
            now_ist      = datetime.now(IST)
            started_utc  = parse_dt_utc(row["startedAt"])
            started_ist  = to_ist(started_utc)
            
            # Get workflow info
            wf_id        = row["workflowId"]
            wf_name      = row["workflowName"] or "unknown"

            # Get owner identity from cached map
            wf_info      = workflow_map.get(wf_id, {})
            owner_email  = wf_info.get("owner_email",  "unknown")
            owner_name   = wf_info.get("owner_name",   "unknown")
            project_name = wf_info.get("project_name", "unknown")

            # Detect new workflows
            is_new_workflow = wf_id not in seen_workflows
            if is_new_workflow:
                seen_workflows[wf_id] = {
                    "name":       wf_name,
                    "first_seen": now_ist.isoformat()
                }

            # Build enriched event
            event = {
                "timestamp":       now_ist.isoformat(),
                "integration":     "n8n",
                "exec_id":         row["id"],
                "workflow_id":     wf_id,
                "workflow_name":   wf_name,
                "exec_status":     row["status"],
                "mode":            row["mode"],
                "finished":        bool(row["finished"]),
                "started_at_utc":  row["startedAt"],
                "started_at_ist":  started_ist.strftime("%Y-%m-%d %H:%M:%S IST") if started_ist else None,
                "stopped_at_utc":  row["stoppedAt"],
                "duration_sec":    duration,
                "retry_of":        row["retryOf"] if row["retryOf"] else "",
                "is_new_workflow": is_new_workflow,
                "owner_email":     owner_email,
                "owner_name":      owner_name,
                "project_name":    project_name,
            }
            
            # Write JSON event to log
            lf.write(json.dumps(event) + "\n")
            last_id = row["id"]

    # Save state
    save_cursor(last_id)
    save_seen_workflows(seen_workflows)

# ═══════════════════════════════════════════════════════════
# Main Loop
# ═══════════════════════════════════════════════════════════

if __name__ == "__main__":
    os.makedirs(os.path.dirname(CURSOR_F), exist_ok=True)
    print("n8n Wazuh poller started (IST)")
    while True:
        try:
            poll()
        except Exception as e:
            print(f"[ERROR] {e}")
        time.sleep(INTERVAL)

Make executable:

sudo chmod +x /opt/YOUR_SCRIPTS_PATH/poller.py

Script 2: heartbeat.py — Dead Silence Detector

Purpose: Runs hourly via cron. If no executions occurred in the past 2 hours, writes a synthetic dead_silence alert. This solves the fundamental problem that event-driven systems cannot detect the absence of events.

Save as: /opt/YOUR_SCRIPTS_PATH/heartbeat.py

#!/usr/bin/env python3
"""
n8n → Wazuh Integration: Heartbeat Script
Detects dead silence (no executions in past 2 hours) and injects synthetic alert.

BEFORE RUNNING: Update these path variables for your environment!
"""
import sqlite3, json, os
from datetime import datetime, timezone, timedelta

# ═══════════════════════════════════════════════════════════
# PATH CONFIGURATION — UPDATE THESE FOR YOUR ENVIRONMENT
# ═══════════════════════════════════════════════════════════

DB_PATH  = "/opt/n8n-server/YOUR_N8N_DATA_PATH/database.sqlite"  # ← UPDATE THIS
LOG_PATH = "/var/log/n8n-executions.log"                          # ← UPDATE THIS

IST = timezone(timedelta(hours=5, minutes=30))

# ═══════════════════════════════════════════════════════════
# Silence Detection
# ═══════════════════════════════════════════════════════════

def check_silence():
    """Check if any executions occurred in past 2 hours"""
    now_ist  = datetime.now(IST)
    cutoff   = now_ist - timedelta(hours=2)
    cutoff_str = cutoff.strftime("%Y-%m-%d %H:%M:%S")

    con = sqlite3.connect(f"file:{DB_PATH}?mode=ro", uri=True)
    count = con.execute(
        "SELECT COUNT(*) FROM execution_entity WHERE startedAt > ?",
        (cutoff_str,)
    ).fetchone()[0]
    con.close()

    if count == 0:
        # No executions - write alert
        event = {
            "timestamp":    now_ist.isoformat(),
            "integration":  "n8n",
            "event_type":   "dead_silence",
            "message":      "No workflow executions detected in the past 2 hours",
            "window_hours": 2,
        }
        with open(LOG_PATH, "a") as lf:
            lf.write(json.dumps(event) + "\n")
        print(f"[ALERT] Dead silence detected — 0 executions in past 2 hours")
    else:
        print(f"[OK] {count} executions in past 2 hours — system active")

if __name__ == "__main__":
    check_silence()

Script 3: summary.py — 12-Hour Health Report Generator

Purpose: Runs at 10:00 and 22:00 IST daily. Produces a rich JSON event containing: total execution counts, success/failure split, per-workflow breakdown, compute time, longest/shortest execution, and full failure list.

Save as: /opt/YOUR_SCRIPTS_PATH/summary.py

#!/usr/bin/env python3
"""
n8n → Wazuh Integration: Summary Report Script
Generates 12-hour health reports with aggregate statistics.

BEFORE RUNNING: Update these path variables for your environment!
"""
import sqlite3, json, os
from datetime import datetime, timezone, timedelta

# ═══════════════════════════════════════════════════════════
# PATH CONFIGURATION — UPDATE THESE FOR YOUR ENVIRONMENT
# ═══════════════════════════════════════════════════════════

DB_PATH  = "/opt/n8n-server/YOUR_N8N_DATA_PATH/database.sqlite"  # ← UPDATE THIS
LOG_PATH = "/var/log/n8n-executions.log"                          # ← UPDATE THIS

IST = timezone(timedelta(hours=5, minutes=30))

# ═══════════════════════════════════════════════════════════
# Helper Functions
# ═══════════════════════════════════════════════════════════

def parse_dt(val):
    """Parse datetime string"""
    if not val:
        return None
    for fmt in ("%Y-%m-%d %H:%M:%S.%f", "%Y-%m-%d %H:%M:%S",
                "%Y-%m-%dT%H:%M:%S.%fZ", "%Y-%m-%dT%H:%M:%SZ"):
        try:
            return datetime.strptime(val, fmt)
        except:
            continue
    return None

def calc_duration(started, stopped):
    """Calculate duration in seconds"""
    s = parse_dt(started)
    e = parse_dt(stopped)
    if s and e:
        return round((e - s).total_seconds(), 2)
    return 0.0

# ═══════════════════════════════════════════════════════════
# Summary Generation
# ═══════════════════════════════════════════════════════════

def generate_summary():
    """Generate 12-hour summary report"""
    now    = datetime.now(IST)
    cutoff = now - timedelta(hours=12)
    cutoff_str = cutoff.strftime("%Y-%m-%d %H:%M:%S")

    # Query all executions in 12hr window
    con = sqlite3.connect(f"file:{DB_PATH}?mode=ro", uri=True)
    con.row_factory = sqlite3.Row
    rows = con.execute("""
        SELECT e.id, e.workflowId, w.name AS workflowName,
               e.status, e.mode, e.finished,
               e.startedAt, e.stoppedAt, e.retryOf
        FROM execution_entity e
        LEFT JOIN workflow_entity w ON w.id = e.workflowId
        WHERE e.startedAt > ?
        AND e.deletedAt IS NULL
        ORDER BY e.startedAt ASC
    """, (cutoff_str,)).fetchall()
    con.close()

    # Initialize counters
    total_executions  = len(rows)
    total_success     = 0
    total_failed      = 0
    total_retries     = 0
    total_duration    = 0.0
    longest_exec      = {"workflow_name": None, "exec_id": None, "duration_sec": 0}
    shortest_exec     = {"workflow_name": None, "exec_id": None, "duration_sec": float("inf")}
    workflow_stats    = {}
    failed_workflows  = []
    executions_detail = []

    # Process each execution
    for row in rows:
        wf_name  = row["workflowName"] or "unknown"
        duration = calc_duration(row["startedAt"], row["stoppedAt"])
        status   = row["status"]

        total_duration += duration

        # Count by status
        if status == "success":
            total_success += 1
        elif status == "error":
            total_failed += 1
            failed_workflows.append({
                "exec_id":       row["id"],
                "workflow_name": wf_name,
                "started_at":    row["startedAt"],
                "stopped_at":    row["stoppedAt"],
                "duration_sec":  duration,
                "retry_of":      row["retryOf"],
            })

        if row["retryOf"]:
            total_retries += 1

        # Per-workflow stats
        if wf_name not in workflow_stats:
            workflow_stats[wf_name] = {
                "total": 0, "success": 0, "failed": 0,
                "total_duration_sec": 0.0, "avg_duration_sec": 0.0
            }
        workflow_stats[wf_name]["total"]              += 1
        workflow_stats[wf_name]["total_duration_sec"] += duration
        if status == "success":
            workflow_stats[wf_name]["success"] += 1
        elif status == "error":
            workflow_stats[wf_name]["failed"]  += 1

        # Track longest/shortest
        if duration > longest_exec["duration_sec"]:
            longest_exec = {
                "workflow_name": wf_name,
                "exec_id": row["id"],
                "duration_sec": duration
            }
        if duration > 0 and duration < shortest_exec["duration_sec"]:
            shortest_exec = {
                "workflow_name": wf_name,
                "exec_id": row["id"],
                "duration_sec": duration
            }

        # Store execution detail
        executions_detail.append({
            "exec_id":       row["id"],
            "workflow_name": wf_name,
            "status":        status,
            "mode":          row["mode"],
            "started_at":    row["startedAt"],
            "stopped_at":    row["stoppedAt"],
            "duration_sec":  duration,
            "retry_of":      row["retryOf"],
        })

    # Calculate averages
    for wf in workflow_stats:
        t = workflow_stats[wf]["total"]
        if t > 0:
            workflow_stats[wf]["avg_duration_sec"] = round(
                workflow_stats[wf]["total_duration_sec"] / t, 2)
        workflow_stats[wf]["total_duration_sec"] = round(
            workflow_stats[wf]["total_duration_sec"], 2)

    if shortest_exec["duration_sec"] == float("inf"):
        shortest_exec = {"workflow_name": None, "exec_id": None, "duration_sec": 0}

    # Build summary event
    summary = {
        "timestamp":          now.isoformat(),
        "integration":        "n8n",
        "event_type":         "12hr_summary",
        "summary_from":       cutoff.isoformat(),
        "summary_to":         now.isoformat(),
        "window_hours":       12,
        "total_executions":   total_executions,
        "total_success":      total_success,
        "total_failed":       total_failed,
        "total_retries":      total_retries,
        "total_duration_sec": round(total_duration, 2),
        "total_duration_min": round(total_duration / 60, 2),
        "avg_duration_sec":   round(total_duration / total_executions, 2) if total_executions else 0,
        "longest_execution":  longest_exec,
        "shortest_execution": shortest_exec,
        "workflow_breakdown": workflow_stats,
        "failed_executions":  failed_workflows,
        "executions":         executions_detail,
    }

    # Write to log
    with open(LOG_PATH, "a") as lf:
        lf.write(json.dumps(summary) + "\n")

    print(f"[OK] Summary written — {total_executions} executions, "
          f"{total_success} success, {total_failed} failed, "
          f"total time: {round(total_duration/60, 2)} min")

if __name__ == "__main__":
    generate_summary()

Script 4: workflow_map_builder.py — Owner Identity Cache

Purpose: Runs every 48 hours via cron. Queries a 5-table join to build a complete map of workflow_id → owner email/name/project. Saves to workflow_map.json. The poller reads this file for O(1) enrichment instead of repeated DB joins.

Save as: /opt/YOUR_SCRIPTS_PATH/workflow_map_builder.py

#!/usr/bin/env python3
"""
n8n → Wazuh Integration: Workflow Map Builder
Builds cached map of workflow_id → owner identity to avoid expensive joins.

BEFORE RUNNING: Update these path variables for your environment!
"""
import sqlite3, json, os
from datetime import datetime, timezone, timedelta

# ═══════════════════════════════════════════════════════════
# PATH CONFIGURATION — UPDATE THESE FOR YOUR ENVIRONMENT
# ═══════════════════════════════════════════════════════════

DB_PATH  = "/opt/n8n-server/YOUR_N8N_DATA_PATH/database.sqlite"  # ← UPDATE THIS
MAP_PATH = "/opt/YOUR_SCRIPTS_PATH/workflow_map.json"             # ← UPDATE THIS

IST = timezone(timedelta(hours=5, minutes=30))

# ═══════════════════════════════════════════════════════════
# Map Building
# ═══════════════════════════════════════════════════════════

def load_existing_map():
    """Load existing map if present"""
    try:
        return json.load(open(MAP_PATH))
    except:
        return {}

def build_map():
    """Build complete workflow → owner identity map"""
    existing = load_existing_map()
    now_ist  = datetime.now(IST).isoformat()

    con = sqlite3.connect(f"file:{DB_PATH}?mode=ro", uri=True)
    con.row_factory = sqlite3.Row

    # 5-table join to get workflow ownership
    rows = con.execute("""
        SELECT
            w.id          AS workflow_id,
            w.name        AS workflow_name,
            w.active      AS is_active,
            w.createdAt   AS created_at,
            u.email       AS owner_email,
            u.firstName   AS owner_first,
            u.lastName    AS owner_last,
            p.name        AS project_name,
            p.type        AS project_type
        FROM workflow_entity w
        LEFT JOIN shared_workflow sw ON sw.workflowId = w.id
        LEFT JOIN project p ON p.id = sw.projectId
        LEFT JOIN project_relation pr ON pr.projectId = p.id
        LEFT JOIN user u ON u.id = pr.userId
        WHERE (p.type = 'personal' OR p.type IS NULL)
        AND (w.isArchived = 0 OR w.isArchived IS NULL OR w.isArchived = 'false')
        ORDER BY w.createdAt ASC
    """).fetchall()
    con.close()

    print(f"Found {len(rows)} workflow rows from DB")

    new_map      = {}
    new_count    = 0
    update_count = 0

    for row in rows:
        wf_id      = row["workflow_id"]
        owner_fn   = row["owner_first"] or ""
        owner_ln   = row["owner_last"]  or ""
        owner_name = f"{owner_fn} {owner_ln}".strip() or "unknown"

        entry = {
            "workflow_name":  row["workflow_name"] or "unknown",
            "is_active":      bool(row["is_active"]),
            "created_at":     row["created_at"],
            "owner_email":    row["owner_email"] or "unknown",
            "owner_name":     owner_name,
            "project_name":   row["project_name"] or "unknown",
            "project_type":   row["project_type"] or "unknown",
            "map_updated_at": now_ist,
        }

        if wf_id not in existing:
            new_count += 1
            entry["first_mapped_at"] = now_ist
            print(f"  [NEW]     {entry['workflow_name']} -> {entry['owner_email']}")
        else:
            entry["first_mapped_at"] = existing[wf_id].get("first_mapped_at", now_ist)
            old = existing[wf_id]
            if old.get("owner_email") != entry["owner_email"] or \
               old.get("workflow_name") != entry["workflow_name"]:
                update_count += 1
                print(f"  [UPDATED] {entry['workflow_name']} -> {entry['owner_email']}")

        new_map[wf_id] = entry

    # Save map
    with open(MAP_PATH, "w") as f:
        json.dump(new_map, f, indent=2)

    total = len(new_map)
    print(f"\n[OK] Map built — {total} workflows total | {new_count} new | {update_count} updated")
    print(f"[OK] Saved to {MAP_PATH}")
    print(f"[OK] Timestamp: {now_ist}")

    # Show stats
    owners = {}
    for wf in new_map.values():
        email = wf["owner_email"]
        owners[email] = owners.get(email, 0) + 1

    print("\n--- Workflows per owner ---")
    for email, count in sorted(owners.items(), key=lambda x: -x[1]):
        print(f"  {email}: {count} workflow(s)")

if __name__ == "__main__":
    print(f"Building workflow map from {DB_PATH}")
    build_map()

Technical Challenges Solved

Challenge 1: IST Timezone Bug

Problem: n8n stores all timestamps in UTC. The original poller calculated hour_of_day from raw UTC strings, causing alerts like "workflow ran at hour 3" when it was actually 8:30 AM IST.

Root Cause: Python's datetime.strptime() creates timezone-naive datetime objects by default. Without explicit timezone attachment, all calculations were in naive UTC.

Fix:

# BEFORE (broken)
started_utc = parse_dt_utc(row["startedAt"])  # Returns naive datetime
hour = started_utc.hour  # UTC hour, not IST

# AFTER (fixed)
def parse_dt_utc(val):
    # ... parsing logic ...
    return datetime.strptime(val, fmt).replace(tzinfo=timezone.utc)  # Attach UTC

def to_ist(dt_utc):
    IST = timezone(timedelta(hours=5, minutes=30))
    return dt_utc.astimezone(IST)  # Convert to IST

started_utc = parse_dt_utc(row["startedAt"])
started_ist = to_ist(started_utc)
hour_ist = started_ist.hour  # Correct IST hour

Lesson: Always attach timezone info when parsing timestamps. Naive datetimes cause silent bugs.

Challenge 2: Rule 86600 Silent Swallowing

Problem: All n8n events were being matched and suppressed by Wazuh's built-in rule 86600 ("Suricata messages") at level 0, preventing all custom rules from ever firing.

Discovery Process:

# Ran manual log test
sudo /var/ossec/bin/wazuh-logtest

# Pasted sample n8n event, saw Phase 3 output:
**Phase 3: Completed filtering (rules).
       Rule id: '86600'
       Level: '0'
       Description: 'Suricata messages.'

Rule 86600 has level="0" and a generic decoder that matches any JSON with certain patterns. It was catching n8n events first.

Fix: Created overwrite rules that intercept 86600 specifically for n8n events:

<rule id="86601" level="0" overwrite="yes">
  <if_sid>86600</if_sid>
  <field name="integration">^n8n$</field>
  <description>Suricata message (overridden for n8n integration)</description>
</rule>

Lesson: Use wazuh-logtest to debug rule evaluation chains. Overwrite rules can prevent generic catch-alls from swallowing specific events.

Challenge 3: Decoder Prematch Whitespace Mismatch

Problem: Custom decoder with <prematch>{"integration":"n8n"}</prematch> (no spaces) never matched actual log output of "integration": "n8n" (space after colon).

Root Cause: JSON serialization in Python adds spaces after colons by default: json.dumps(event) produces "integration": "n8n", not "integration":"n8n".

Fix:

<!-- BEFORE (broken) -->
<prematch>{"integration":"n8n"}</prematch>

<!-- AFTER (fixed) -->
<prematch>"integration": "n8n"</prematch>

Lesson: Match exact serialization format. Test with actual log output, not hand-written JSON.

Challenge 4: User Identity Without Direct Foreign Key

Problem: n8n's execution_entity table has no userId column. How do we know which analyst triggered a specific execution?

Solution: Traced the full ownership chain across 5 tables:

execution_entity 
  → workflow_entity (via workflowId)
  → shared_workflow (via workflowId) 
  → project (via projectId)
  → project_relation (via projectId)
  → user (via userId)

Since all 6 analysts in our SOC work in personal projects (not team projects), the workflow owner is effectively the triggering user for manual executions.

Implementation:

  • workflow_map_builder.py runs this 5-table join every 48 hours
  • Caches result as JSON map: workflow_id → {owner_email, owner_name, project_name}
  • poller.py does O(1) dict lookups instead of repeated 5-table joins

Lesson: When direct foreign keys don't exist, trace entity relationships. Cache expensive joins.

Final Thoughts

If you're running n8n or any automation platform in your SOC and it's not feeding into your SIEM — you have a blind spot.

Your SIEM monitors threats. Your automation responds to threats. But who monitors the automation itself?

This integration gives you that missing piece. Every execution. Every failure. Every manual trigger. Every analyst. Complete visibility.

Why This Matters — The Conceptual Point Wazuh → n8n is about response. A threat is detected, n8n automates the action.

n8n → Wazuh is about accountability and visibility over the automation layer itself.

Without this integration, your SIEM monitors your infrastructure but is completely blind to the system that responds to your infrastructure alerts. If that system fails silently, your automated SOC response stops — and your SIEM has no record of it happening.

This integration closes that loop.
